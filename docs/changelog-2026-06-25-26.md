## Feature Summary (User-Facing)

### TV Rename — Episode Intelligence *(new, 2026-06-26)*

| Feature | What It Does |
|---|---|
| **Wrong-episode detection** | When the matched episode's runtime is suspicious (>10% off), ScanHound scans ±3 episodes in the same season and ±1 adjacent seasons to find a better runtime match. If a candidate scores 15+ confidence points higher, it surfaces as a **suggested correction** in `needs_review` status — never auto-applied. |
| **Ollama tie-breaker** | When two episode candidates are within 10 points of each other, the local Ollama model picks between them using the filename and episode metadata. Falls back silently if Ollama is unavailable. |
| **Combined multi-episode detection** | If a file's runtime is roughly 1.7–2.4× a single episode and matches the sum of two consecutive episodes within 8%, it is flagged as a **combined episode** (e.g. two eps aired together). Surfaces as a proposal in `needs_review`. |
| **Split-file detection** | If a file's runtime is under 0.6× the expected episode length and a sibling `Part 1`/`Part 2` file exists in the same folder, both are flagged as a **split pair**. Surfaces as a proposal in `needs_review`. |
| **Multi-episode naming** | Files correctly identified as multi-episode are named in Plex convention: `Show (2024) - S01E01E02 - Title.mkv`. Previously only single-episode format was supported. |
| **Split-file naming** | Split files get a `- Part 1` / `- Part 2` suffix: `Show (2024) - S01E03 - Title - Part 1.mkv`. |
| **Download-page hints** | When a download record has page text, ScanHound extracts early signals (combined/split/part number/episode count) via Ollama with a regex fallback. Stored on the download record as an additional signal. |
| **Runtime neutral zone tightened** | Runtime confidence scoring now treats ≤10% deviation as neutral (was ≤15%). A 13-minute variance on a 90-minute episode no longer scores the same as a perfect match. |

> **All detections are proposals only.** Any file with a suggested correction, combined-episode flag, or split-file flag is forced to `needs_review` and cannot be auto-renamed. Human confirmation is always required.

### TV & Movie Rename — Identification Pipeline *(new/improved, 2026-06-25/26)*

| Feature | What It Does |
|---|---|
| **Runtime cross-check confidence signal** | The matched title's TMDB runtime is now compared against the actual file runtime (via `ffprobe`) and feeds into the confidence score — improving or penalising based on how well they align. |
| **Episode validity check** | Before accepting a TV match, ScanHound verifies the parsed episode number actually exists in the matched season on TMDB. Invalid episode numbers reduce confidence instead of passing silently. |
| **Video-frame vision identification** | Tier 5 fallback: if all text-based tiers fail, ScanHound extracts frames from the video and passes them to the local vision model (Ollama `minicpm-v`) for identification. |
| **Progressive frame scan** | Frame extraction samples priority positions (intro, act break, ending) first, then fills in a grid if those frames are blank/black. Avoids cold-open misidentification. |
| **File-size plausibility fallback** | When `ffprobe` is unavailable, a file-size estimate is used in place of the runtime check so confidence scoring degrades gracefully rather than failing. |
| **Plain-text IMDb fallback** | If the structured TMDB search returns no confident match, ScanHound fetches the IMDb plain-text search page as an additional signal before falling back to LLM tiers. |
| **Ollama context enrichment** | During LLM-tier identification, available page text and filename context are passed to the local Ollama model to improve match quality before the vision fallback is attempted. |

### Mobile / Android *(new, 2026-06-25)*

| Feature | What It Does |
|---|---|
| **Android app** | Native Android client for ScanHound (Tauri v2 mobile build). Browse and manage rename jobs from Android. |
| **Swipe deck UI** | Card-swipe interface for mobile: swipe right to approve a rename, left to skip, up to open details. Faster than the list view on a phone. |
| **Mobile UI** | Responsive layout across the SvelteKit frontend tuned for small screens. |
| **Multi-select genre & language filters** | Filter the job queue by multiple genres or languages simultaneously. Previously only single-value filtering was supported. |
